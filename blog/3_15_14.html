Fourier Transforms¶

Fourier transforms are pretty cool. Today we are going to discuss two import aspects of them.

First of all, they are useful in modeling periodic functions.

If we have a function \(f(x)\) then the fourier transform is defined as

\[A_{k} = \int_{-\infty}^{\infty} f(x) e^{-i\pi k} dx\]

From this definition, we can build an approximation of \(f(x)\) in the following way

\[f(x) = A_{0} + A_{1}e^{i\pi} + A_{2}e^{2i\pi} + A_{3}e^{3i\pi} + ...\]

Obviously the more terms included in this expansion, the better the model will be.

from scipy.integrate import quad #this is just standard integration
def fourier(function,k,a=0,b=2*pi):
    """hmmm... assuming the range is from 0 to 2*pi.
    function should be some kind of lambda like function
    k is the kth coefficient, which is supposedly what will be returned to you 
    """
    fn = lambda x: function(x)*exp(1j*k*x*2*pi/(b-a))
    real_part = lambda x: real(fn(x)) #sp.real()
    imag_part = lambda x: imag(fn(x))
    return 1/(b-a)*quad(real_part,a,b)[0]+1j/(b-a)*quad(imag_part,a,b)[0]

Now let's use this fourier transform algorithm to model the sawtooth wave.

The second aspect of Fourier transforms that I want to look at today is that the Fourier transform allows us to look at our function in frequency space instead of in position space.

This is really cool, because it gives us a second, alternative way to look at our data and this may give us more insight into what we are examining.

Instead of working with the sawtooth wave, let us work with a gaussian (normal) curve. A gaussian curve has the important property that the fourier transform of a gaussian is still a gaussian.

Let's check this out.

So let's go ahead and learn to program the FFT algorithm ourselves.

From wikipedia: http://en.wikipedia.org/wiki/Fast_Fourier_transform and http://en.wikipedia.org/wiki/Cooley-Tukey_FFT_algorithm

The DFT (discrete fourier transform) is defined by the formula

\[X_{k} = \sum\limits_{n=0}^{N-1}x_{n}e^{-2\pi ikn/N} \hspace{12 mm} k = 0, 1, ..., N-1\]

We can then seperate the odd and even terms

\[X_{k} = \sum\limits_{n=0}^{N/2-1}x_{2n}e^{-2\pi ik(2n)/N} + \sum\limits_{n=0}^{N/2-1}x_{2n+1}e^{-2\pi ik(2n+1)/N}\]

\[ = x_{n} + w_{n}y_{n}\]

where

\[ x_{n} = \sum\limits_{n=0}^{N/2-1}x_{2n}e^{-2\pi ikn/(N/2)} ; \hspace{2 mm} y_{n} = \sum\limits_{n=0}^{N/2-1}x_{2n+1}e^{-2\pi ikn/(N/2)}; \hspace{2 mm} and \hspace{2 mm} w_{n} = e^{-2\pi ik/N}\]

As far as I can tell, this so far doesn't actually give us any speed boost. What does give us a speed boost is the observation that we actually only need to calculate the first half of these terms because

\[ X_{k} = x_{n} + w_{n}y_{n} \hspace{12 mm} for \hspace{2 mm} n < N/2 \hspace{2 mm} and\] \[ X_{k+N/2} = x_{n} - w_{n}y_{n} \hspace{12 mm} for \hspace{2 mm} n < N/2 \]

I believe this is where we get the big speed up.

def myFFT(l):
    L =len(l)
    if L == 1:
        return l
    elif L%2 == 0:
        out = [0 for i in xrange(L)]
        
        for i in xrange(L/2):
            A = myFFT(l[::2])
            B = myFFT(l[1::2])
            out[i] = (A[i] + B[i]*exp(-2j*pi*i/L))
            out[i+L/2] = (A[i] - B[i]*exp(-2j*pi*i/L))
        return out
    else:
        raise IndexError("powers of 2 please")
